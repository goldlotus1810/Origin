## III. Vấn Đề Cốt Lõi

### 1. RESPONSE TEMPLATE QUÁ ĐƠN GIẢN (Nghiêm trọng nhất)
HomeOS không có khả năng sinh ngôn ngữ tự nhiên. Toàn bộ response dựa trên ~10 câu template cố định trong `response_template.rs`. Dù emotion pipeline phân tích chính xác V/A/tone, kết quả cuối cùng vẫn chỉ là chọn 1 trong 10 câu.

**Hệ quả:** Người dùng nói bất cứ gì → nhận được 1 trong ~5 câu giống nhau → cảm giác "không hoạt động".

### 2. COMMAND PARSING BỊ HẠN CHẾ
`is_command()` trong parser.rs chỉ nhận: dream, stats, health, seed, shutdown, reboot, status, help + math commands + leo commands.

THIẾU: typeof, explain, why, trace, inspect, assert — tất cả các lệnh debug/reasoning.

Nhưng `handle_command()` trong origin.rs CÓ code xử lý typeof/explain/why. Vấn đề là parser không route chúng đến đó — chúng bị parse thành text thường hoặc query, không phải Command.

### 3. 82% NODES KHÔNG ĐƯỢC PHÂN LOẠI
SystemManifest dựa trên alias name patterns để classify. 202/246 nodes = "Uncat" (unclassified). NodeKind metadata tồn tại nhưng SystemManifest không đọc nó.

### 4. AGENT LOOP LÀ CODE CHẾT
```rust

// Nhưng workers rỗng → không có Worker→Chief message
// Chiefs không tự generate message → Chief→LeoAI = 0
```

### 5. SILK GRAPH TĂNG NHƯNG KHÔNG ẢNH HƯỞNG
Silk tăng từ 128→134 nodes qua mỗi phiên. Edges co-activate đúng. Nhưng kết quả walk_weighted() không ảnh hưởng đến response text vì response chỉ là template.
